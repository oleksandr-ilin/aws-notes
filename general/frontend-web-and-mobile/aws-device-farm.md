Overview
Test mobile and web apps on real devices in the cloud.

Tasks
- Upload apps, run test suites on device pools, and review results.

Alternatives
| Alternative | Main pro | Main con |
|---|---:|---|
| Local device labs | Full control | Hardware & maintenance costs |
| Third‑party device farms | Different device coverage | Integration differences |

Limitations
- Device availability and concurrency limits; test execution quotas.

Price info
- Per-device-minute or remote access fees depending on mode.

Network & multi-region
- Tests run in AWS regions; network conditions may differ from user environments.

Popular use cases
- Compatibility testing, regression test suites, and automated CI integration.

When Not to Use
- Very small projects where manual testing suffices.

DR strategy
- Archive test reports and device logs to S3 for auditing.
