Overview
Discovers on‑prem servers and applications to build inventory and dependency maps for migrations.

Tasks
- Run discovery agents or connectors, collect inventory, and analyze application dependencies.

Alternatives
| Alternative | Main pro | Main con |
|---|---:|---|
| Manual discovery | No tooling cost | Time-consuming and error-prone |
| Third-party discovery tools | Richer integrations | Cost & integration effort |

Limitations
- Agent coverage and firewall/network access may limit visibility.

Price info
- Charged per-agent or per-scan depending on mode and scale.

Network & multi-region
- Data exported to a region; plan secure connectivity for agent reporting.

Popular use cases
- Migration assessments, dependency mapping for rehosting or replatforming.

When Not to Use
- Very small migrations where manual inventory is sufficient.

DR strategy
- Archive discovered inventories and dependency graphs for validation during cutover.
